Database Technologies: SQL vs. NoSQL

  • Database technology overview
  • SQL vs NoSQL
  • How apps store data

Why we need databases

  • Apps need memory to store data
  • Persistant data storage is required
  • Fast, safe, organized

SQL

Relational DB - stores data in structured tables with rows and columns, and relationships are enforced using keys.

  • + Relational DB = Structured DB
  • + Fixed schema - tables, rows, columns
  • - Hard to scale

NoSQL

Non-relational DB - stores data in flexible formats like documents, key-value pairs, or graphs, without strict table relationships.

  • + Non-relational DB = flexible and scalable
  • + No strict schema = scalable and can handle unstructured data
  • - Weaker consistence

SQL vs NoSQL

SQL

  • Structure
  • Consistency
  • Precision
  • Best for structured business data (login data, payments, etc)

NoSQL

  • Flexibility
  • Scalability
  • Speed
  • Best for rapidly changing data (user-generated contenct, chat messages, logs, etc)

SQL (PostgreSQL) example


import { Pool } from 'pg';

const pool = new Pool({
  host: 'localhost',
  user: 'postgres',
  password: 'yourpassword',
  database: 'myapp',
  port: 5432,
});

export const db = pool;

async function createUsersTable() {
  await db.query(`
    CREATE TABLE IF NOT EXISTS users (
      id SERIAL PRIMARY KEY,
      email TEXT UNIQUE NOT NULL,
      name TEXT NOT NULL
    `);
  )
}

SQL (PostgreSQL) example


async function addUser(email: string, name: string) {
  const result = await db.query(
    'INSERT INTO users (email, name) VALUES ($1, $2)',
    [email, name]
  );
}

addUser('alice@example.com', 'Alice').catch(console.error);
  

async function getUsers() {
  const result = await db.query(
    'SELECT id, email, name FROM users'
    );
  return result;
}

getUsers().catch(console.error);

SQL Top commands


-- Select all columns
SELECT * FROM users;

-- Filter rows
SELECT * FROM users WHERE name = 'Alice';

-- Limit number of rows
SELECT * FROM users LIMIT 5;

-- Sort results ascending / descending
SELECT * FROM users ORDER BY name ASC;
SELECT * FROM users ORDER BY name DESC;

-- Update rows
UPDATE users SET name = 'Bob' WHERE id = 1;

-- Insert new row
INSERT INTO users (name, email) VALUES ('Charlie', 'charlie@example.com');

-- Delete rows
DELETE FROM users WHERE id = 2;

NoSQL (MongoDB) example


import { MongoClient } from 'mongodb';

const client = new MongoClient(process.env.MONGO_URL!);
await client.connect();

const db = client.db('myapp');
const users = db.collection('users');
  

async function addUser(email: string, name: string) {
  await users.insertOne({
    email: email,
    name: name,
    createdAt: new Date(),
  });
}

addUser('alice@example.com', 'Alice').catch(console.error);
  

async function getUsers() {
  const allUsers = await users.find().toArray();
  return allUsers;
}

getUsers().then(console.log).catch(console.error);

NoSQL Top Commands (MongoDB)


// Select all documents
db.users.find();

// Filter documents by field
db.users.find({ name: 'Alice' });

// Limit number of results
db.users.find().limit(5);

// Sort results ascending / descending
db.users.find().sort({ name: 1 });  // ASC
db.users.find().sort({ name: -1 }); // DESC

// Update documents
db.users.updateOne(
  { _id: ObjectId("...") },
  { $set: { name: 'Bob' } }
);

// Insert new document
db.users.insertOne({ name: 'Charlie', email: 'charlie@example.com' });

// Delete documents
db.users.deleteOne({ _id: ObjectId("...") });

Combining SQL and NoSQL

  • Both used at the same time in large systems
  • Right tool for specific task
